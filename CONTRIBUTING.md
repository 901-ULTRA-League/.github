ðŸ›  Development Workflow
1. Fork the Repository

Create a fork of the project you want to contribute to and clone it locally.

git clone https://github.com/<your-username>/<repo-name>.git

2. Create a Feature Branch

Use a descriptive name that matches the change:

git checkout -b feature/add-deck-filter

3. Commit Using Clear Messages
git commit -m "Added advanced deck filtering"


Follow conventional commit principles when possible (e.g., feat:, fix:, docs:).

4. Push and Open a Pull Request
git push origin feature/add-deck-filter


Then open a PR against the appropriate branch (typically main).

ðŸ“‘ Pull Request Guidelines

Before submitting a Pull Request:

Ensure your code passes all linting and formatting checks

Add tests where applicable

Update documentation if your change introduces new behavior

Reference related Issues (e.g., Resolves #42)

Provide a concise summary of what changed and why

All major changes should be discussed in an Issue before development begins to avoid duplicate work.
